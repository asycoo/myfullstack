# 全栈实战学习计划（博客 / C 端内容产品）

**适用人群**：前端基础较好，后端初学者，希望通过实战成为可独立交付的全栈。  
**技术主线**：Next.js（App Router）+ Prisma + PostgreSQL（与当前仓库一致）。  
**时间投入**：每周约 **6 小时**。  
**产品方向**：**A. 公众号 / 博客型**（可读、可分享、可被发现）。

---

## 学习目标（一年后“自主全栈”的可验收定义）

你能独立完成一个中小型内容产品从 **需求 → 数据建模 → 接口 → 前端 → 安全基线 → 部署上线 → 排障** 的闭环，并且：

- 能解释你的数据表为什么这么设计、索引为什么这样加。
- 能处理常见线上问题（迁移、环境变量、依赖不可用、性能退化）。
- 有固定回归手段（`lint`、脚本式 `smoke`、或最小 E2E）。

> 说明：你当前仓库已完成大量“后端基础设施”练习（鉴权、分层、CSRF、限流、密码策略、日志脱敏、部署）。本计划从 **第 1 周起聚焦“内容站产品化”**，把后端能力嵌进真实用户路径里学。

---

## 每周固定节奏（6 小时怎么花）

| 时长 | 做什么 |
| --- | --- |
| 3h | 写功能（必须可运行、可提交） |
| 1.5h | 读官方文档 / 排错（只解决本周卡点） |
| 1h | 写复盘（建议仍放在 `DOC/`，固定模板见文末） |
| 0.5h | 回归：`npm run lint` + `BASE_URL=... npm run test:smoke`（按需扩展 smoke） |

**原则**：每周只推进 **一个主题**，避免并行需求导致学不到底。

## 12 周课表（博客 / 内容站）

> 路由建议：**第 1 周先用数字 `id` 做详情**（学习曲线平滑），**第 3 周再上 `slug`**（更接近真实博客）。

### 第 1 周：公开文章详情（读者链路）

- **目标**：未登录也能阅读已发布文章；复制链接可在无痕窗口打开。
- **后端练习点**：区分「公开读」与「写接口需要登录」；404/403 语义。
- **验收**：无痕窗口打开详情页成功；未发布内容不可公开访问。

### 第 2 周：列表只展示已发布内容

- **目标**：公开列表默认只返回 `published=true`（或等价字段）。
- **后端练习点**：查询条件、分页参数（可先保留现有列表，再加 query）。
- **验收**：草稿不会出现在公开列表；作者仍可在“我的后台”看到草稿（可先简单处理）。

### 第 3 周：`slug` 与漂亮 URL

- **目标**：`Post.slug` 唯一；创建/编辑时生成；详情支持 `/posts/[slug]`。
- **后端练习点**：唯一约束、冲突（409）、字符串规范化（trim、小写、非法字符）。
- **验收**：`/posts/hello-world` 可访问；重复 slug 有稳定错误结构。

### 第 4 周：摘要 `excerpt` + 封面 `coverImage`（封面可先 URL）

- **目标**：列表更像内容站；详情展示完整信息。
- **后端练习点**：可选字段、默认值、Zod 校验边界。
- **验收**：列表展示摘要/封面；缺省时不崩。

### 第 5 周：SEO：`title` / `description`（Metadata）

- **目标**：详情页 `metadata` 使用文章标题与摘要。
- **练习点**：理解“查看源代码里 meta 正确”为验收标准。
- **验收**：页面源代码中 meta 信息正确。

### 第 6 周：`sitemap.xml`

- **目标**：输出站点地图，包含文章 URL。
- **后端练习点**：稳定 URL 规则、聚合查询、limit。
- **验收**：访问 `/sitemap.xml` 能看到文章链接。

### 第 7 周：`robots.txt`

- **目标**：允许抓取并指向 sitemap。
- **验收**：`/robots.txt` 内容正确。

### 第 8 周：RSS（`/rss.xml`）

- **目标**：输出最近 N 篇文章的 RSS。
- **后端练习点**：排序、limit、XML 转义（避免注入/坏字符）。
- **验收**：RSS 阅读器可订阅或浏览器打开为合法 XML。

### 第 9 周：站内搜索 v1（标题/摘要模糊查询）

- **目标**：`GET /api/posts/search?q=` + 简单前端搜索框。
- **后端练习点**：`ILIKE`、分页、索引的局限（先能用再优化）。
- **验收**：能搜到标题关键词；空查询有合理行为。

### 第 10 周：阅读时间 `readingTime`

- **目标**：保存或更新文章时估算阅读分钟数并展示。
- **后端练习点**：派生字段、更新策略（编辑后重算）。
- **验收**：详情页展示阅读时长；与字数变化一致。

### 第 11 周：分享卡片（Open Graph 基础）

- **目标**：最少实现 `openGraph.title/description`（图片可先固定一张或后续再做动态图）。
- **验收**：用分享调试工具能看到 OG 字段（图片可先简化）。

### 第 12 周：性能小复盘（一次“像工程师”的优化）

- **目标**：挑一个慢查询或列表接口，加索引 + `EXPLAIN`，把结论写入 `DOC/`。
- **后端练习点**：性能必须有证据，不是感觉优化。
- **验收**：一篇笔记能说清楚：慢在哪里、改了什么、如何验证变快。

## 第 1 周「6 小时」按小时排程（可直接照做）

| 小时 | 做什么 | 产出 |
| --- | --- | --- |
| 1 | 画用户路径：读者如何打开文章 | 1 张纸/一段文字：URL、是否需要登录 |
| 2 | 增加详情 API：`GET /api/posts/:id`（仅 published） | 接口可用（可用 curl/浏览器验证） |
| 3 | 增加详情页：`/posts/[id]`（公开可读） | 页面能渲染 |
| 4 | 列表页加跳转：标题链接到详情 | 体验闭环 |
| 5 | 处理边界：404、未发布、作者预览（可先不做预览） | 行为稳定 |
| 6 | 回归 + 复盘文档 | `lint` + `smoke` + `DOC` 记录 |
